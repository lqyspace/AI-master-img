## 灰度发布系统协议设计

> **数据协议**

- 定长Header

  ```json
  {
      uid,
      token,
      IP,
      tag,
      mtest, // 全链路压测使用
      cmd,
      sessionID,
      bodyLen
  }
  ```

- 变长Body

  ```json
  {
      实际传输的内容
  }
  ```

- 设计初确定灰度发布字段

  - uid
  - Token
  - IP
  - Tag

> **灰度发布策略**

- 单策略

  - uid/token/IP

    假设对uid%100取模，得到的值可以决定是否进行灰度发布

- 基于上下文灰度策略

  - 多个模块A/C同时灰度
  - tag
